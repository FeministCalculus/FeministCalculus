# CALIBRATE-SWARM-04: ℋ_k(1) Bootstrap 稳健性检验报告

**生成日期**：2026-06-09  
**Fc 审计版本**：v9.6.7-REVISED 兼容  
**检验对象**：ℋ_k(1) 触发条件 = (TFR < 2.1) ∧ (d²TFR/dt² < −0.05)  
**面板数据**：11 国 × 2017-2024 TFR（World Bank A 级来源）

---

## 一、检验设计

### 检验 1：Leave-One-Year-Out (LOYO)

**目的**：检验 ℋ_k(1) 触发事件是否对单年数据删除敏感。  
**方法**：删除每一年（2017-2024），用剩余年份重新计算所有可能的二阶导数，观察原始触发事件是否消失。  
**区分**：
- **平凡丢失**：删除触发年份本身导致丢失（中心差分需要该年数据，必然丢失）。
- **非平凡丢失**：删除**非触发年份**导致触发事件丢失（表明事件对相邻年份数据敏感）。
- **非平凡稳健**：删除任何非触发年份都不丢失。

### 检验 2：TFR Perturbation Bootstrap

**目的**：检验 ℋ_k(1) 触发事件对 TFR 测量误差的稳健性。  
**方法**：对 TFR 值加入随机噪声（正态分布，σ ∈ {0.01, 0.02, 0.03, 0.05}），重复 1000 次，观察：
- 原始触发事件的复现率
- 噪声导致的假阳性事件数
- 信号强度分类（|d²| / 0.05）

## 四、Bootstrap 检验结果

### 原始事件复现率

| 事件 | σ=0.01 | σ=0.02 | σ=0.03 | σ=0.05 | 信号强度 |
|---|---|---|---|---|---|
| 中国 2019 | **100.0%** | **100.0%** | **99.3%** | **93.4%** | 4.3× (强) |
| 德国 2021 | **100.0%** | **99.4%** | **95.8%** | **83.3%** | 3.5× (强) |
| 瑞典 2021 | **100.0%** | **95.9%** | **88.6%** | **76.3%** | 2.8× (中等) |
| 法国 2022 | 79.3% | 62.8% | 58.5% | 56.2% | 1.4× (弱) |
| 法国 2021 | 66.0% | 60.4% | 54.3% | 54.1% | 1.2× (弱) |

### 假阳性分析（σ=0.02）

| 指标 | 数值 |
|---|---|
| 原始触发事件数 | 5 |
| Bootstrap 均值 | 11.7 |
| Bootstrap 标准差 | 2.2 |
| Bootstrap 范围 | [5, 20] |
| 假阳性增量 | +6.7 事件/运行（134%） |

**解读**：σ=0.02 的 TFR 噪声（约 ±0.02 的测量误差）导致平均每次运行产生 11.7 个触发事件，其中 6.7 个为噪声制造的假阳性。这表明 ℋ_k(1) 的阈值 −0.05 对测量误差敏感，存在**假阳性膨胀**风险。

## 六、阈值敏感性分析

当前阈值 d² < −0.05 的设定导致：
- **法国 2021**（d² = −0.060）和 **法国 2022**（d² = −0.070）处于阈值边缘。
- 微小的测量误差（σ = 0.02）即可使复现率降至 60%。
- 假阳性膨胀显著（+134%）。

**建议**：引入**信号强度分层阈值**：

| 信号强度 | 判定 | 阈值 |
|---|---|---|
| |d²| / 0.05 > 3.0 | **ℋ_k(1) 触发**（高置信） | d² < −0.15 |
| 1.5 < |d²| / 0.05 ≤ 3.0 | **ℋ_k(1) 预警**（中等置信） | −0.15 ≤ d² < −0.075 |
| 1.0 < |d²| / 0.05 ≤ 1.5 | **ℋ_k(0) 观察**（低置信） | −0.075 ≤ d² < −0.05 |
| |d²| / 0.05 ≤ 1.0 | 不触发 | d² ≥ −0.05 |

按此分层：
- **中国 2019**（4.3×）→ ℋ_k(1) 触发（高置信）
- **德国 2021**（3.5×）→ ℋ_k(1) 触发（高置信）
- **瑞典 2021**（2.8×）→ ℋ_k(1) 预警（中等置信）
- **法国 2022**（1.4×）→ ℋ_k(0) 观察（低置信）
- **法国 2021**（1.2×）→ ℋ_k(0) 观察（低置信）

## 七、对 TIFM 框架的修正建议

### 1. ℋ_k 分层体系

将单一的 ℋ_k(1) 阈值扩展为**三层信号体系**：

```
ℋ_k(2) = (TFR < 2.1) ∧ (d² < −0.15)   # 高置信触发
ℋ_k(1) = (TFR < 2.1) ∧ (d² < −0.075)  # 中等置信预警  
ℋ_k(0) = (TFR < 2.1) ∧ (d² < −0.05)   # 低置信观察
```

### 2. 假阳性控制

- 报告 ℋ_k 触发时，必须同时报告**信号强度**（|d²| / 0.05）。
- 信号强度 < 1.5 的事件，标注为"观察级"，不进入级联判定。
- 信号强度 ≥ 3.0 的事件，标注为"触发级"，可直接进入级联分析。

### 3. 数据质量要求

- ℋ_k 触发判定要求 TFR 数据来源为 **A 级**（World Bank / 各国统计局）。
- 若使用 B 级或 C 级来源，信号强度阈值上调 50%（即 ℋ_k(2) 要求 d² < −0.225）。

## 八、关键节点状态变量更新

- **固化参数**：ℋ_k(2/1/0) 分层阈值（新增）

- **活跃风险**：
