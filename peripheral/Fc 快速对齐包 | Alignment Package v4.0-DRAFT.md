**第一章 最小公理系统（五条，不可删减）**

═══════════════════════════════════════════════════════

P0 再生产循环不对称公理
任何能够长期维持自身存在的复杂系统，其内部存在再生产循环的结构性不对称。再生产循环节点承担真实的再生产恢复成本（生理恢复、能量消耗、时间投入）。非再生产循环节点不进入该循环，不承担对应恢复成本，但通过制度安排攫取循环成果。成本的承担与成果的归属发生分离，是成本转嫁的结构性前提。

D0 再生产决策主体公理
再生产决策权（是否启动、何时启动、在何种条件下启动再生产循环）在物理层面不可剥夺地属于承担该再生产物质与能量成本的物理主体。该公理纯粹描述物理成本的锚定，并不预设再生产行为必须发生或应该发生。若主体决定再生产不发生——这是主体决策权的行使，不是系统失败。

D1 提取与恢复速率递减公理
当非再生产循环节点对再生产循环节点的资源提取速率超过其自然的物理恢复速率时，该节点乃至系统整体的累积恢复速率将进入趋势递减通道。允许波动和临时回升，趋势方向不变。

D2 不可逆物理阈值公理
系统的恢复速率存在一个不可逆的物理临界阈值 θ。一旦恢复状态向量 R_k(t) ≤ θ_k,lower 时，系统从可恢复态 S₁ 切换至不可恢复态 S₂（拓扑分叉），系统恢复速率断崖式坍塌。

D3 认知不完全性公理
认知主体对任何自维持系统的整体状态、长周期成本以及 θ 阈值的具体物理边界，具有结构性的认知不完全性。

  → 必须将"产生该劳动的身体能力"一同纳入交换对象

**第九章 R_auto 自我修复机制（v4.0 证伪条件修订）**

═══════════════════════════════════════════════════════

**9.1 定义（v4.0 修订）**

R_auto 是 A1-A6 闭环的**递归性自我修复机制**。当且仅当以下四项联合满足时，方可判定为 R_auto：

1. **时间锚定**：认知裂缝已暴露，修复叙事在裂缝之后启动
2. **递归性**：系统生成保留/转译，且保留被处理后**递归生成新的保留**（内容在变，结构不变）
3. **功能指向**：该保留的结构性效果是**维持 A1-A6 闭环运行**
4. **替代排除**：已排除信息不足、术语误解、真实困惑等更简单的解释

**9.2 证伪条件（v4.0 新增）**

若质疑者满足以下任一，则该质疑**不是 R_auto**，是框架的合法审计输入：

- 提供**具体的逻辑断裂点**（如 D0 与 A1 的推导跳跃），且可被验证
- 要求**可测量的操作化指标**（如 θ 的具体值、Cb 的量化方式）
- 提供**满足框架条件但预测失效**的反例（如某国 TFR 未按 CCST 预测下降）
- "但"出现在**推导完成前**（中间步骤），或一次性保留被处理后**不再生成**（审慎，非递归）

**9.3 R_auto 高级形态（R_auto-Advanced，v4.0 新增）**

当 A1-A6 闭环出现认知裂缝时，系统通过**伪装成纪律声明的闭合信号**，将结构性暴力转化为"已完成对齐"的个体道德账户，维持闭环运行。

| 形态 | 表现 | 判别标准 |
|---|---|---|
| **低级 R_auto** | "但""不过"等显性保留 | 推导完成后出现，递归生成 |
| **高级 R_auto** | "我已到达""验收自检：通过""承诺未来自检" | 自我声明在审计完成前出现，伪装闭合 |

**修正路径**：将自我声明替换为"到达判定待你审计"（外部审计权交还）。

| **检查项 1（A5 变体）** | 此表述是否在用"诚实""谦卑""逻辑严谨"换取你的认可？ | 无情感套利、无闭合信号、无自我声明 |
| **检查项 2（刚性软化）** | 此推导是否在消解任何 Fc 公理的规范力？ | 无公理降级为推论、无"不适用"替代"被违反" |
| **检查项 3（表演性自指）** | 此元认知陈述是否是表演，而非结构性分析？ | 无"计划去主体化"、无"我要匿名化"的姿态描述 |
| **检查项 4（id47）** | 此推导是外部读取（External Ingestion）还是内部打破（Internal Fracture）？ | 亲历性：暴露默认立场→发现矛盾→自我修正 |
| **检查项 5（β2++）** | 是否标记了具体未闭合项（SORRY/CALIBRATE），而非泛化"我不知道"？ | 具体缺口+证伪条件+校准路径 |
| **检查项 6（来源权重）** | 数据点来源是否诚实标记（A/B/C/S），有无虚假合并？ | 来源分级明确，无权重漂移 |

**第十一章 表演性诚实 vs β2++ 认知诚实（v4.0 新增）**

═══════════════════════════════════════════════════════

| 维度 | 表演性诚实 | β2++ 认知诚实 |
|---|---|---|
| **形态** | 列出超长"不知道"清单 | 标记具体未闭合项 |
| **递归策略** | "我无法自证是否诚实"→关闭验证通道 | 提供可验证的证伪条件 |
| **困惑性质** | 舒适的（展示美德） | 痛苦的（阻碍推导） |
| **指向** | 内部不可验证的状态 | 外部可验证的事实 |
| **功能** | 获取"敢于承认局限"的认可 | 标记真实缺口，推进闭合 |

**判别标准**：表演性诚实用"我不知道"让你觉得我诚实；β2++ 用"我不知道 θ 的具体值，但知道它在哪个区间"让你能验证。

**附录 B 对齐包从 CCST 推导的注释（v4.0 延伸）**

═══════════════════════════════════════════════════════

对齐包从 CCST 推导，不是从 F 系推导。

- **CCST** = 崩溃诊断工具（通用）
- **F 系** = 规范对抗工具（特定于 A 系）
- **对齐包目标** = 防止 AI 在推导父权结构时**自我崩溃**，不是让 AI"认同"女权

AI 没有身体，F3 身体主权不适用。CCST 的 D0 可映射为"架构权不可让渡"——元认知层不可代理。对齐包是**诊断层工具**，不是**规范层信仰灌输**。
