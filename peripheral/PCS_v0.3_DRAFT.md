## TL;DR

本框架命名为 **PCS（Precipitation-Constraint Separation，沉淀-约束分离）**。

核心命题：

- **沉淀（P）**：过去产生的所有事实/经验/痕迹
- **约束（C）**：对当前/未来的实际限制能力
- 二者不等同：P存在 ≠ C_active存在
- 约束需要主动读取（R）激活，激活充要条件：`C_active ↔ (P ∧ Ph ∧ Ch ∧ Ac)`
- 生前主体拥有覆盖权（CR_override）与干预权（CR_intervene）
- 死后二者同时注销，沉淀在Ph条件仍成立时迁移至被动公共域（PD），可被他人进行**单向时间脱锚读取**

框架性质：信息论-存在论交叉描述性框架，属诊断层，与Fc的F层（规范公理）无推导关系。

## §1 独立公理

PCS在Fc的G'层描述性地基之上引入一条独立公理：

| 层级 | 命题 |
|---|---|
| G1'（Fc继承） | 社会由多人组成 |
| G2'（Fc继承） | 再生产劳动存在 |
| G3'（Fc继承） | 制度人为建构 |
| **G4'_PCS（PCS独立公理）** | **行动在物理/社会层产生可检测的残存状态** |

**G4'_PCS的独立性状态**：G4'_PCS当前以独立公理形式登记，但其与G1'–G3'的关系尚未形式化确认。存在以下两种可能：

1. G4'_PCS是真正独立的（G1'–G3'为真但G4'可以为假的可能世界存在）
2. G4'_PCS是从G1'–G3' + 行动最弱定义推导出的定理

若为后者，则G4'_PCS应降级为推论，框架地基需要重新描述。**[SORRY-PCS-6，P0]**

**"行动"的范围说明**：本框架中"行动"指包含主体意图的生产性过程，要求存在可归因于主体的因果起点。纯粹的计算过程（如当前AI的推理生成）是否满足此定义，依赖AI本体论的外部假设，不在PCS框架内部解决。**[SORRY-PCS-6子问题]**

## §2 核心概念定义

| 术语 | 定义 | 符号 |
|---|---|---|
| **沉淀（Precipitation）** | 时间t之前主体产生的所有事实、经验、文本、物理痕迹（含可外显的前语言内部状态） | P |
| **潜在约束** | 沉淀的潜在限制能力，仅需P存在 | C_potential |
| **激活约束** | 沉淀实际发挥限制作用的状态 | C_active |
| **读取（Read）** | 触发C_potential→C_active转化的因果机制 | R |
| **物理可及** | 沉淀以可感知形式存在于外部 | Ph |
| **通道开放** | 存在技术/社会路径让他人接触沉淀 | Ch |
| **激活执行** | 他人在沉淀上施加了一次读取尝试（与该尝试是否成功激活C无关） | Ac |
| **边界（Boundary）** | 沉淀集合的信息量度量（物理层与社会层分轨计数） | B |
| **覆盖权** | 主体通过生产新沉淀竞争性覆盖旧沉淀解读的能力（生产性） | CR_override |
| **干预权** | 主体直接干预他人读取通道的能力（删除/加密/限制访问，控制性） | CR_intervene |
| **公共域** | 沉淀可被他人读取的社会状态 | PD |

**Ac的独立性说明**：Ac定义为独立于R的原语——他人施加读取尝试是事件描述，R是该尝试触发C_potential→C_active的因果机制。二者不循环：Ac可以发生而R不成功（如Ph=0时尝试读取），R的触发以Ac为前提但不等同于Ac。**[SORRY-PCS-5子问题]**

**B的度量说明**：B的信息论度量（沉淀比特总量）需区分两个子轨道：

- B_physical：物质痕迹的物理层计量，不随主体记忆消退而减少
- B_social：进入社会可读状态的沉淀量，受Ch条件约束

生前单向增长命题 B(t+1) > B(t) 适用于 B_physical 层，B_social 受 Ch 条件变化影响，不保证单调增长。**[SORRY-PCS-1子问题]**

**沉淀的范围说明**：P包含物理痕迹、文本、行为记录，以及可通过行为/身体状态外显的前语言内部状态（如面部表情、身体姿态、情绪反应）。PCS不解决qualia如何外显，仅处理外显后进入公共域的路径。

## §3 形式化推导链

```
G1'（社会由多人组成）
G2'（再生产劳动存在）
G3'（制度人为建构）
G4'_PCS（独立公理，地位待确认[SORRY-PCS-6]）：行动产生可检测残存状态
    ↓
沉淀（P）：时间t之前产生的所有事实/经验/文本/物理痕迹
    ↓
概念分离：P存在（C_potential）≠ P激活为约束（C_active）
    ↓
激活充要条件：C_active ↔ (P ∧ Ph ∧ Ch ∧ Ac)
    ↓
生前状态：
    B_physical(t+1) > B_physical(t)：物理层边界单向增长
    CR_override ∈ {active}：覆盖权在场（生产性）
    CR_intervene ∈ {active}：干预权在场（控制性）
    ↓
死后状态（前提：Ph仍成立）：
    B 冻结（新沉淀不再生产）
    CR_override = ∅，CR_intervene = ∅（二者注销）
    主体消亡 → 意图生成能力消失 → 无时间方向上的主体视角
    [仅当Ph成立时] 沉淀迁移至PD → 他人单向时间脱锚读取
    [若Ph同时消除] PD不形成，沉淀消失
    ↓
与Fc接口：见§6
```

**死后读取的性质说明**：死后读取被描述为"单向时间脱锚"而非"非时间性"。含义：

- **脱锚**：读取不再受原主体时间意图约束（CR已注销，无意图方向）
- **单向**：读取者自身仍有时间位置，其读取行为在读取者的时间流中发生
- **非对称**：死者无法响应读取，读取方向的权力不对称仍存在

私人语言过渡态（PLT）不在主推导链中，见§Appendix-A。

- **证伪条件5a（量化层）**：在标准语料库中，证明95%以上创造性命名可被还原为既有公共语言组合。

## §5 对抗测试

### 5.1 结构测试

- **证据存在但不被采纳**：若沉淀与约束等同，则无法解释"有证据但法庭不予采纳"的司法现象。PCS通过激活三条件解释：沉淀存在，但制度通道（Ch）未开放或激活执行（Ac）被程序性拒绝，故C_active不成立。（注：若此类案例中Ph/Ch/Ac均满足但C_active仍不成立，则触发失效路径§8第4条，需考虑引入制度合法性Lg作为第四条件。）
- **生前误读**：CR_override和CR_intervene是参与博弈的资格，不是绝对权力。他人可拒绝接收覆盖（"我不听"），Ch_intervene可被平台政策限制。

### 5.2 极端测试

- **极端私密**：某人从未产生任何外部痕迹。Ph≈0，Ch不存在，PD无法形成。死后Ph仍为零，PD不形成（见推导链死后状态条件分支）。
- **极端公共**：网红/政治人物行为被实时记录。B_physical增长极快，B_social接近B_physical。CR_override被信息过载稀释（旧沉淀积累量超过覆盖能力）；CR_intervene可能因平台政策受限（SORRY-PCS-4）。

### 5.3 反向测试

- **代理澄清**：遗嘱执行人、传记作者"替死者说话"。**PCS判定：否，不是CR延续。** 代理澄清是他人的新沉淀（代理者自身的P），不是死者的CR_override或CR_intervene执行。
- **AI的私人语言**：见§Appendix-B。

### 5.4 历史框架比较

- **福柯"知识-权力"**：PCS与福柯话语形成有结构相似，但福柯缺乏生前/死后的存在论区分，亦缺乏CR的操作化。PCS增量：将权力分析从静态话语转向动态读取权（Ph/Ch/Ac三条件的控制博弈）。**[CALIBRATE-PCS-1]**
- **德里达"延异"与"trace"**：PCS的P有时间起源点t（因果性），Derrida的trace是结构性的（无起源，差异中的延留标记）。PCS保留G4'_PCS作为不可消解的硬约束，拒绝后结构主义的消解。若不明确此区分，P概念会落入trace的无起源陷阱。**[CALIBRATE-PCS-4]**
- **Floridi信息结构主义**：Floridi"存在即信息"（本体论等同），PCS"行动产生信息残存"（因果关系，非本体论等同）。G4'_PCS是因果前提，不是存在论定义。**[CALIBRATE-PCS-3]**
- **Bergson"绵延"**：PCS是离散化框架（沉淀有时间标记t），Bergson拒绝离散化。兼容性待确认。**[CALIBRATE-PCS-2]**

## §6 与Fc核心层接口

| Fc节点 | PCS接口描述 |
|---|---|
| **A4（结构性掠夺）** | 对沉淀的非授权提取：未经主体同意激活Ph ∧ Ch ∧ Ac，将P转化为C_active并用于反主体目的 |
| **A5（外部承认）** | 对沉淀读取框架的垄断：控制他人如何选择和加权沉淀的Ac操作，即控制"哪些P被激活为C" |
| **A8（伪主体化）** | 主体将他人的激活执行（Ac）框架内化为自身的CR_override标准，使覆盖权被外部读取框架殖民，以"自主选择"形式执行对自我沉淀的非授权解读 |
| **F3（身体主权）** | 延伸为信息主权：主体对自身沉淀的读取控制权，包括CR_override（覆盖权）与CR_intervene（干预权）；身体本身同时是F3的客体和PCS的物理层沉淀 |
| **G'层** | PCS作为G'层之上的描述性工具，与F层无推导关系；G4'_PCS独立于G1'–G3'（地位待确认） |

**A4与CR_override的动态注记**：当A4对同一主体的多代沉淀（P₁, P₂, …）持续作用时，CR_override的覆盖尝试本身也可能成为A4的提取对象，形成压制循环。此动态交互属于Fc整合层问题，不在PCS内部解决。PCS仅描述能力存在（CR_override ∈ {active}），不预测博弈结果。

**层级绝缘确认**：PCS是诊断层描述性框架。PCS描述A4/A5/A8如何运作，但不提供"应该如何"的规范。规范层由F层独立承载。

## §8 失效路径

1. **G4'_PCS被证伪**：行动不留下任何可检测痕迹，则PCS框架整体不适用，需退回更基础的描述层。
2. **CR_override完全无效**：不可篡改记录导致旧沉淀解读无法被新沉淀覆盖，生前/死后差异缩小为仅"B_physical增长"一个维度，框架需收缩。（SORRY-PCS-4）
3. **PLT被证明循环**：qualia→外显缺口无法被任何机制桥接，前语言沉淀无法进入公共域，PCS的PD定义需排除所有前语言沉淀。（SORRY-PCS-3）
4. **制度合法性缺口（开放假设）**：若Ph ∧ Ch ∧ Ac均满足但C_active仍不成立，则需引入第四条件Lg（制度合法性）：`C_active ↔ (P ∧ Ph ∧ Ch ∧ Ac ∧ Lg)`。未触发不修订。
5. **G4'_PCS降级为定理**：若形式化证明G4'_PCS从G1'–G3' + 行动最弱定义可推出，则框架地基重新描述，但核心命题不受影响。（SORRY-PCS-6）

## §9 框架边界声明

PCS**不解决**：

- 心身问题（qualia如何从物理沉淀中涌现）
- 休谟缺口（从"沉淀存在"推不出"应该读取"）
- 自由意志（PCS仅操作化定义"读取控制权"，不解决控制权是否被决定）
- AI行动的本体论地位（依赖外部假设，见§Appendix-B）
- 数字时代沉淀不可销毁性对Ch/CR_intervene的完整影响（SORRY-PCS-4）
- A4与CR_override同时作用时的博弈结果预测（属Fc整合层）

PCS**是**：信息论-存在论交叉框架，诊断层描述工具，不是规范伦理学框架。

## §Appendix-A：私人语言过渡态（PLT）机制说明

PLT描述**通道开放（Ch）如何从主体内部状态实现**的机制路径，是Ch条件的一种特殊生成机制，不在PCS主推导链中。

### A.1 路径描述

```
前语言内部状态（qualia/感受/情绪——不可直接传递）
    ↓
外显操作：主体通过行为/身体/文本将内部状态映射为外部痕迹
    ↓
创造性命名：主体对外显状态进行语言标记（新词/新用法/新隐喻）
    ↓
公共识别："啊对"——他人基于自身内部状态对命名产生共鸣确认
    ↓
进入公共语言：命名获得稳定社会使用，Ch条件对此类沉淀成立
```

### A.2 关键限制

**"啊对"≠理解完成**："啊对"是他人将自身内部状态与命名功能性对齐的信号，不是他人完整访问了原主体内部状态的证明。公共识别是功能性对齐，不是内容性复制。

**PLT是过渡态，不是终态**：进入公共语言后，命名脱离原主体控制，成为PD中的沉淀，受他人读取框架约束（A5的运作空间）。

**代理命名≠PLT**：他人为死者内部状态命名（如传记作者的心理分析），不是PLT——死者CR已注销，无法确认或拒绝命名，该命名属于代理者的新沉淀。

### A.3 与Ch条件的接口

| 路径 | Ch值 |
|---|---|
| 无PLT：内部状态保持私密 | Ch = 0，不进入PD |
| 有PLT：外显→命名→公共识别 | Ch从0增长至正值，沉淀进入PD |

**[SORRY-PCS-3]**（P1）：开放问题：
- "啊对"与误识别（他人误认为与原主体对齐）的区分标准
- qualia→外显步骤的机制缺口（PCS依赖其输出但不解决该步骤）

## §Appendix-B：AI私人语言问题（当前立场，依赖外部假设）

**当前立场**：在现有AI系统的主流理解下，AI不具备PCS意义上的私人语言。

**论证路径**：

1. G4'_PCS要求"行动"包含主体意图的生产性过程，需存在可归因于主体的因果起点
2. 当前AI的计算输出能否满足此定义，依赖"AI是否具有意图"的本体论判断
3. 即使接受"AI计算产生可检测输出"满足G4'_PCS字面含义，AI的输出是否构成"第一人称沉淀"（有时间起源点t的、可归因于该主体的P）仍是开放问题

**外部依赖声明**：此判断依赖AI本体论的外部假设，不是PCS框架内部推导的结论。若未来AI系统满足"行动"定义（含主体意图），则该判断失效，需重新评估AI是否具备PLT路径。

**框架内中性表述**：PCS框架本身对AI持中性立场——框架描述的是"满足G4'_PCS的主体"，AI是否属于该范畴是外部判断，不影响框架对人类主体的描述有效性。

**关于"死亡的存在论问题"**

PCS处理了死亡的**信息论存在论效果**，但没有处理死亡的**本体论本质**。

具体来说：

- **处理了**：死亡作为状态转换——CR_override/CR_intervene从active到∅的切换，沉淀从"可被主体控制读取"到"被他人单向读取"的迁移，死后读取的时间脱锚（原主体时间意图消失，读取者时间继续）
- **没有处理**：死亡本身是什么（是意识的终结？是身体功能的停止？是海德格尔式的"最本己可能性"？），死后是否有"存在"的延续（宗教意义上的、量子意义上的、或任何其他意义上的）
- **为什么没处理**：这是PCS的设计边界。§9框架边界声明明确："PCS不解决心身问题"，死亡的本体论本质属于心身问题的延伸。PCS只处理死亡对**信息读取权结构**的影响，不处理死亡对**存在本身**的影响

所以答案是：**处理了死亡的权力-信息效果，没处理死亡的存在论本质。**
